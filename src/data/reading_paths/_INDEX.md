## AI Agent Instructions: Knowledge-Structure Update

When processing a new paper, determine whether it belongs in any reading path:

1. **Default: Don't add.** Reading paths contain only genuinely foundational papers — not every good study.
2. **Only proceed if ALL of these apply:**
   - The paper is foundational to its domain (not just another experiment in a long series)
   - It advances the narrative of a specific path at a particular point
   - It is not redundant with an already-listed paper
   - It is cited by multiple subsequent sources or appears in major meta-analyses
3. **DO NOT read all path files.** Use the Domain-to-Path Mapping table above to identify the 1-2 relevant paths, then Grep the relevant path file for the paper's author before opening it.
4. **If adding:** Match the existing entry format exactly. Renumber all subsequent entries.

## How to use these paths

- **Read actively**: for each paper, note the year, the journal, and the sample size or scope. These contextual details matter as much as the conclusions.
- **Cross-reference paths**: many papers appear in more than one path. Each time you encounter a paper in a different context, you will understand it differently.
- **Follow the debate pairs**: several paths deliberately place a pro-psi paper next to its skeptical response. Reading both back-to-back is where the real learning happens.
- **Use the catalog**: every entry includes a catalog ID that maps directly to `_CATALOG.json` for metadata, DOIs, and file locations.
